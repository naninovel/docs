# サポート

助けが必要な場合、まずは公式Discordサーバーに参加することをお勧めします：[discord.gg/BfkNqem](https://discord.gg/BfkNqem)

`#forum` チャンネルで助けやアドバイスを求めたり、`#wiki` でチュートリアルやガイドを探したり、`#chat` で他のNaninovelユーザーと雑談したりすることができます。

有効な [サポートプラン](/ja/support/#サポートプラン) をお持ちの場合は、Naninovelチームが直接支援を提供する `#support` チャンネルにもアクセスできます。サポートプランの最初の1年間は、[ライセンスを登録](https://account.naninovel.com) すると無料になります。

## サポートプラン

サポートプランは、Discordサーバーでの専用サポートへのアクセスを提供し、以下を含みます：

- `#support` チャンネルでの技術的な問題やトラブルシューティングリクエストへの優先的な対応
- 開発プロセスを追跡できるGitHub上のエンジンソースコードリポジトリへのアクセス
- 最新のプレビューおよび安定リリースブランチを含むUPMリポジトリへのアクセス（GitHubから直接Naninovelをインストールおよび更新可能）
- 高度なサンプルとデモシーンを含むプロジェクトへのアクセス

[ライセンスを登録](https://account.naninovel.com) すると、無料の1年間のサポートプランが含まれます。その後は、[アカウントダッシュボード](https://account.naninovel.com/support) からいつでもサブスクリプションを更新できます。

::: info NOTE
サポートプランは完全にオプションです。それがなくても、[ダウンロードアーカイブ](https://account.naninovel.com/download) を介して将来のすべての安定したNaninovelリリースへの生涯アクセス権を引き続き持ちます。また、コミュニティサポートは `#forum` チャンネルで引き続き利用可能であり、いつでも質問して助けを得ることができます。
:::

## バグの報告

意図したとおりに機能していないと思われる場合は、`#support` チャンネル（有効な [サポートプラン](/ja/support/#サポートプラン) をお持ちの場合）または `#forum` チャンネルで [Discord](https://discord.gg/BfkNqem) でお知らせください。

レポートを提出する前に、以下を行ってください：

- 問題が発生している機能またはユースケースについて、[ガイド](/ja/guide/)、[コマンドリファレンス](/ja/api/)、および [FAQ](/ja/faq/) を確認してください。何かを見落としている可能性があります。
- 最新の利用可能なNaninovelバージョンを実行していることを確認してください。最新のパッチは [UPMレジストリ](/ja/guide/getting-started#githubからインストール) から入手できます。アセットストアやダウンロードアーカイブで配布されているパッケージは古い場合がよくあります。
- 最近、以前のNaninovelリリースからアップグレードした場合は、[リリースノート](/releases/) のアップグレード手順に従っていることを確認してください。
- プロジェクトルートの `Library` フォルダを削除してエディタを再起動し、Unityのキャッシュをクリアしてみてください。
- 問題が実際にNaninovelに起因するものであり、他のサードパーティプラグインやUnity自体に起因するものでないことを確認してください。その場合は、[Unityサポートにお問い合わせください](https://unity.com/support-services)。

バグを報告するときは：

- 問題とそれを段階的に再現する方法について、明確かつ簡潔な説明を提供してください。
- Naninovelバージョン、Unityバージョン、ターゲットプラットフォーム（Android、iOS、WebGLなど）、およびオペレーティングシステム（Windows、macOS、またはLinux）を含めてください。
- 関連するエラーや警告を含む [ログファイル](https://docs.unity3d.com/Manual/LogFiles.html) を添付してください。

## 再現プロジェクト

問題を報告する際、それを再現する小さなUnityプロジェクトの提供を求められる場合があります。
再現プロジェクトは、問題を示すために必要な **最小限の** 変更のみを含む **新しいクリーンなUnityプロジェクト** である必要があります。

以下の手順に従ってください：

1. 新しいUnityプロジェクトを作成します。[サポートされているUnityバージョン](/ja/guide/compatibility#unityバージョン) を使用していることを確認してください。
2. 最新の利用可能なNaninovelバージョンをインストールします。パッケージフォルダ内のものを変更または追加しないでください。変更されたバージョンのパッケージはサポートできません。
3. 問題を再現するために必要なアセットとスクリプトを追加します。既存のプロジェクト全体をコピーしないようにしてください。最初からやり直して、隔離された状態で問題を再現してください。サードパーティのプラグインや不要なコンテンツは避けてください。
4. プロジェクトルートに `repro.txt` ファイルを作成し、段階的な手順と、期待される結果と実際に起こることの簡単な説明を記述します。例えば：
    ```
    1. シーン "Assets/Scenes/SampleScene" を開く。
    2. エディタでプレイモードに入る。
    3. 新しいゲームを開始する。
    4. 行番号15までプレイする。
    5. ゲームを保存してロードする。

    期待される結果: 音楽 "Ambient" が再生されるはず。
    実際の結果: 音楽が再生されない。
    ```
5. プロジェクトサイズを小さくするために `Library` フォルダを削除し、プロジェクトフォルダをzip圧縮します。
6. 再現プロジェクトを要求したNaninovelチームメンバーにDiscordのDMで個人的に共有してください。個人情報や著作権で保護されたアセットが含まれている可能性があるため、公開チャンネルでプロジェクトを共有しないでください。
